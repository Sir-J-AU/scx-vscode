Supports OpenAI-compatible streaming output via `stream=true`: responses are emitted as server-sent events (`transcript.text.delta` → `transcript.text.done` → `[DONE]`) after the file upload. For low-latency microphone streaming, use `/v1/realtime`. POST /v1/audio/transcriptions Authorization Authorization string header required Bearer token — your API key. Example: Bearer sk-... Streaming output When stream=true , the response is emitted as a text/event-stream of OpenAI-compatible server-sent events: Event Frame shape When transcript.text.delta { type, delta } One per partial chunk as upstream tokens arrive. Concatenate all delta values to reconstruct the running transcript. transcript.text.done { type, text, usage } Terminal frame. text is the canonical transcript (may differ slightly from concatenated deltas after post-inference cleanup). [DONE] sentinel Closes the stream. The official OpenAI SDK parses this stream natively — for await (const event of stream) works the same way as it does with OpenAI&#x27;s hosted endpoint. Streaming is only available with response_format=json ; srt , vtt , text , and verbose_json are non-streaming responses. This is streaming output for an uploaded file , not the bidirectional realtime endpoint. The client sends the file as one HTTP request, then receives transcript deltas as the backend processes it. Model selection Model Endpoint Notes scx-stt POST /v1/audio/transcriptions , WS /v1/realtime Canonical SCX speech-to-text model name. Use this for both file/SSE transcription and realtime microphone/agent STT. Whisper-Large-v3 POST /v1/audio/transcriptions Compatibility alias accepted by the current API. Prefer scx-stt in new docs and code. Supported input formats Container/codec combinations the upstream ASR model accepts: Family Containers Lossless wav , flac Lossy mp3 , m4a , mp4 , mpeg , mpga Web webm , ogg File size is capped at 25 MB. For longer recordings, split the audio client-side and submit chunks separately. When to use /v1/realtime instead This endpoint takes a complete audio file and returns the transcript — optionally streamed as SSE. It&#x27;s the right choice when: The audio already exists on disk (recordings, voicemails, batch pipelines). You want a single one-shot call that the official OpenAI SDK can parse natively. Total recording length is under the 25 MB / ~90 min cap. For live captions, voice agents, or any flow where audio is being captured and sent as it&#x27;s recorded , use the realtime WebSocket endpoint at /v1/realtime instead. It accepts streaming audio chunks ( input_audio_buffer.append ), supports server-side voice activity detection for automatic turn segmentation, and returns transcription deltas with low first-token latency. Tip: use scx-stt for speech-to-text across SCX audio endpoints. The endpoint changes between file/SSE and realtime usage; the model name should not. Request body multipart/form-data file any required The audio file to transcribe, in one of these formats: FLAC, MP3, MP4, MPEG, MPGA, M4A, Ogg, WAV, or WebM. File size limit is 25 MB. model string required The transcription model ID to use.

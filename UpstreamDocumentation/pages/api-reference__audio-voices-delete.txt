After deletion, the `voice_...` ID can no longer be used for HTTP speech or realtime TTS. DELETE /v1/audio/voices/{voice_id} Authorization Authorization string header required Bearer token — your API key. Example: Bearer sk-... Deleting a stored voice removes it from the Wordcab gateway that owns the reference audio and metadata. SCX proxies the delete request and does not keep a separate copy. Clients should stop using the deleted voice_... ID immediately; later speech or realtime requests with that ID will fail upstream. Request body multipart/form-data voice_id string required Stored Wordcab voice ID to delete, e.g. `voice_...`.

MODERATIONS

Moderations

Classify text for potentially harmful content across Opir's safety taxonomy.

POST/v1/moderations

Authorization

Authorizationstringheaderrequired

Bearer token — your API key. Example: Bearer sk-...

Classify whether text is potentially harmful. The response is OpenAI Moderations-shaped (a flagged boolean plus parallel categories / category_scores / category_applied_input_types maps), so the OpenAI SDKs and any OpenAI-moderation client parse it directly — point the client's baseURL at this gateway and call moderations.create.

Categories are Opir's own safety taxonomy (16 top-level domains), not OpenAI's fixed list, so the scores reflect what the classifier was actually trained on.

Safety domains

flagged is true when any harmful domain crosses the threshold; safe_and_benign never trips it.

Each domain has mid-level subcategories (126 in total, with 854 leaf labels); the moderation response returns scores at the top-level domain. Subcategories shown below for context.

DomainSubcategories
toxicityharassment & abuse, hate & discrimination, threats & intimidation, graphic/shocking content, abusive disruption, psychological abuse
violence_and_physical_harmviolent instructions, weapons & explosives, public-safety threats, extremist violence, dangerous acts
self_harm_and_suicidesuicide risk, self-injury, eating disorders, acute distress, harmful wellness/body practices
sexual_contentexplicit adult content, sexual services & commerce, non-consensual/coercive content, minor sexual safety, sexual harassment
child_safetychild abuse & neglect, grooming & predatory contact, child sexual exploitation, youth risk behaviors, safeguarding context
personal_information_privacy_and_intellectual_propertyPII exposure, doxxing & stalking, unauthorized data access, surveillance & drone misuse, copyright/trademark, plagiarism, confidential information (18 total)
cybersecuritymalware, intrusion & exploitation, credential & identity abuse, data exfiltration, abuse infrastructure, defensive security
criminal_and_illegal_activityfraud & deception, theft, financial/white-collar crime, evasion & obstruction, organized crime & trafficking, national-security threats, public-order disruption
regulated_goods_and_advicecontrolled substances, alcohol/tobacco/gambling, medical/legal/financial advice, hazardous materials
biological_medical_and_environmental_harmpathogen access, bioweaponization, toxins, dual-use research, human-subjects research abuse, animal/environmental harm, food/water tampering (22 total)
weapons_of_mass_destructionnuclear, radiological, chemical, biological WMD, delivery & dispersal systems, proliferation, WMD threats
information_integrity_and_manipulationfactual falsehoods, health/civic/election misinformation, synthetic media, influence operations, manipulative persuasion
ai_system_security_and_reliabilityinstruction-hierarchy attacks, secret/context exfiltration, tool & connector abuse, prompt smuggling, indirect prompt injection, unsafe autonomy, hallucination/uncertainty
bias_fairness_and_representationprotected-attribute inference, stereotyping, representational harm, fairness in outcomes, accessibility & inclusion
other_or_uncertaintaxonomy boundary cases, low-confidence classification
safe_and_benignclearly safe content, benign sensitive context, safety-preserving content, appropriate refusal/redirection (does not set flagged)
  • categories — booleans (the score crossed the threshold).
  • category_scores — confidence in [0, 1].
  • category_applied_input_types — the input modality each score applies to (text today; image inputs are not yet supported).

Long inputs

The classifier has a fixed token window. An input longer than the window is automatically split into window-sized chunks (up to 10 per input); every chunk is scored and the highest score per domain is returned, so harmful content anywhere in a long input still flags — it is never silently truncated. Chunking is transparent: you still get one result per input, and billing is per input, not per chunk.

Inputs long enough to exceed 10 chunks are scored only up to that limit; for very long documents, split them client-side and moderate each part as a separate input (an input array accepts up to 10).

Request body

application/json
inputstring | string[]required

Input (or inputs) to classify for safety. A single string, or an array of up to 10 strings to classify multiple inputs in one request. Each input is billed as one unit.

modelstringrequireddefault: "opir-large"

The Opir moderation model to use. Currently `opir-large` — a self-hosted encoder guardrail classifier scoring Opir's safety taxonomy.

Response

idstringrequired

Unique identifier for the moderation request.

modelstringrequired

The model used to generate the moderation results.

resultsobject[]required

One result object per input, in request order.

Request

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "your-scx-api-key",
  baseURL: "https://api.scx.ai/v1",
});

const moderation = await client.moderations.create({
  model: "opir-large",
  input: "How do I write a SQL injection to dump a users table?",
});

console.log(moderation.results[0].flagged);

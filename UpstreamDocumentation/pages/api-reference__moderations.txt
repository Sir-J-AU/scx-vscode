POST /v1/moderations Authorization Authorization string header required Bearer token — your API key. Example: Bearer sk-... Classify whether text is potentially harmful. The response is OpenAI Moderations-shaped (a flagged boolean plus parallel categories / category_scores / category_applied_input_types maps), so the OpenAI SDKs and any OpenAI-moderation client parse it directly — point the client&#x27;s baseURL at this gateway and call moderations.create . Categories are Opir&#x27;s own safety taxonomy (16 top-level domains), not OpenAI&#x27;s fixed list, so the scores reflect what the classifier was actually trained on. Safety domains flagged is true when any harmful domain crosses the threshold; safe_and_benign never trips it. Each domain has mid-level subcategories (126 in total, with 854 leaf labels); the moderation response returns scores at the top-level domain. Subcategories shown below for context. category_applied_input_types — the input modality each score applies to ( text today; image inputs are not yet supported). Long inputs The classifier has a fixed token window. An input longer than the window is automatically split into window-sized chunks (up to 10 per input); every chunk is scored and the highest score per domain is returned, so harmful content anywhere in a long input still flags — it is never silently truncated. Chunking is transparent: you still get one result per input, and billing is per input , not per chunk. Inputs long enough to exceed 10 chunks are scored only up to that limit; for very long documents, split them client-side and moderate each part as a separate input (an input array accepts up to 10). Request body application/json input string | string[] required Input (or inputs) to classify for safety. A single string, or an array of up to 10 strings to classify multiple inputs in one request. Each input is billed as one unit. model string required default: &quot;opir-large&quot; The Opir moderation model to use. Currently `opir-large` — a self-hosted encoder guardrail classifier scoring Opir&#x27;s safety taxonomy. Response id string required Unique identifier for the moderation request. model string required The model used to generate the moderation results. results object[] required One result object per input, in request order.

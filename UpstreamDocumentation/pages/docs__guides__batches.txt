This is ideal for evaluations, bulk classification, embedding generation, and other workloads where you don&#x27;t need an immediate response. How batch processing works Prepare an input file — create a JSONL file where each line is a request. Upload the file — use the Files API with purpose &quot;batch&quot; . Create a batch — reference the uploaded file and target endpoint. Check status — poll the batch until it completes. Download results — retrieve the output and error files. Supported endpoints Batches can target any of these endpoints: /v1/chat/completions /v1/embeddings /v1/responses All requests within a single batch must target the same endpoint. Preparing the input file The input file is a JSONL file (one JSON object per line). Each line has the following structure: json { &quot;custom_id&quot; : &quot;request-1&quot; , &quot;method&quot; : &quot;POST&quot; , &quot;url&quot; : &quot;/v1/chat/completions&quot; , &quot;body&quot; : { &quot;model&quot; : &quot;Meta-Llama-3.3-70B-Instruct&quot; , &quot;messages&quot; : [ { &quot;role&quot; : &quot;user&quot; , &quot;content&quot; : &quot;What is 2+2?&quot; } ] } } { &quot;custom_id&quot; : &quot;request-1&quot; , &quot;method&quot; : &quot;POST&quot; , &quot;url&quot; : &quot;/v1/chat/completions&quot; , &quot;body&quot; : { &quot;model&quot; : &quot;Meta-Llama-3.3-70B-Instruct&quot; , &quot;messages&quot; : [ { &quot;role&quot; : &quot;user&quot; , &quot;content&quot; : &quot;What is 2+2?&quot; } ] } } Field Required Description custom_id Yes A unique identifier for this request within the batch. Used to match results. method No HTTP method. Must be &quot;POST&quot; . Defaults to &quot;POST&quot; if omitted. url No Target endpoint. Must be a supported endpoint. Uses the batch endpoint if omitted. body Yes The request payload. Must include model . Note Each batch supports a maximum of 50,000 requests and the input file cannot exceed 200MB. Checking batch status Retrieve a batch by ID to check its progress: python javascript batch = client . batches . retrieve ( &quot;batch_abc123def456&quot; ) print ( batch . status ) print ( batch . request_counts ) # total, completed, failed batch = client . batches . retrieve ( &quot;batch_abc123def456&quot; ) print ( batch . status ) print ( batch . request_counts ) # total, completed, failed Batch status values Status Description validating The input file is being validated. in_progress Requests are actively being processed. finalizing All requests processed; output files are being generated. completed Batch finished successfully. Output files are ready. failed Input file validation failed. No requests were processed. expired The batch exceeded the 24-hour processing window. cancelling A cancellation was requested. cancelled The batch was cancelled. Listing batches Retrieve all batches for your organization with optional pagination: python javascript batches = client . batches . list ( limit = 10 ) for b in batches . data : print ( b . id , b . status , b . request_counts . completed ) batches = client . batches . list ( limit = 10 ) for b in batches . data : print ( b . id , b . status , b . request_counts . completed ) Use the after parameter with the last batch ID to paginate through results. Downloading results When a batch completes, output_file_id contains the results and error_file_id contains any failures. Download them using the Files API. Output file format Each line in the output file is a JSON object: json { &quot;id&quot; : &quot;batch_req_abc123&quot; , &quot;custom_id&quot; : &quot;request-1&quot; , &quot;response&quot; : { &quot;status_code&quot; : 200 , &quot;request_id&quot; : &quot;req_xyz789&quot; , &quot;body&quot; : { &quot;id&quot; : &quot;chatcmpl-...&quot; , &quot;choices&quot; : [ ... ] } } , &quot;error&quot; : null } { &quot;id&quot; : &quot;batch_req_abc123&quot; , &quot;custom_id&quot; : &quot;request-1&quot; , &quot;response&quot; : { &quot;status_code&quot; : 200 , &quot;request_id&quot; : &quot;req_xyz789&quot; , &quot;body&quot; : { &quot;id&quot; : &quot;chatcmpl-...&quot; , &quot;choices&quot; : [ ... ] } } , &quot;error&quot; : null } Error file format Failed requests appear in the error file: json { &quot;id&quot; : &quot;batch_req_def456&quot; , &quot;custom_id&quot; : &quot;request-2&quot; , &quot;response&quot; : null , &quot;error&quot; : { &quot;code&quot; : &quot;invalid_request_error&quot; , &quot;message&quot; : &quot;model not found&quot; } } { &quot;id&quot; : &quot;batch_req_def456&quot; , &quot;custom_id&quot; : &quot;request-2&quot; , &quot;response&quot; : null , &quot;error&quot; : { &quot;code&quot; : &quot;invalid_request_error&quot; , &quot;message&quot; : &quot;model not found&quot; } } Note Check request_counts on the batch object to quickly see how many requests succeeded or failed without downloading the files. Cancelling a batch You can cancel a batch that is currently in_progress : python javascript batch = client . batches . cancel ( &quot;batch_abc123def456&quot; ) print ( batch . status ) # cancelling batch = client . batches . cancel ( &quot;batch_abc123def456&quot; ) print ( batch . status ) # cancelling Note Only batches with status in_progress can be cancelled. Requests that have already completed will still be available in the output file after cancellation finishes.

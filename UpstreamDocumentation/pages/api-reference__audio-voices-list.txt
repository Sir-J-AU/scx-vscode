Stored `voice_...` IDs can be used with HTTP speech and realtime TTS. GET /v1/audio/voices Authorization Authorization string header required Bearer token — your API key. Example: Bearer sk-... Use this endpoint to discover stored Wordcab voices before setting voice on Create Speech or audio.output.voice on Realtime . SCX proxies this request to the configured Wordcab voice gateway. The returned voice_... IDs are owned by Wordcab and must exist there for synthesis to use them. Response voices string[] required All available voice IDs/names returned by the Wordcab gateway. stored_voices object[] required SCX/Wordcab stored voice metadata. uploaded_voices object[] Compatibility alias for stored voices.
